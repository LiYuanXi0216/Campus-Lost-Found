## 第一章 概述

### 1.1 项目背景与目的

Campus Lost-Found 校园失物招领平台是一个面向校园场景的失物招领系统，旨在帮助师生快速有效地发布寻物启事和失物招领信息，解决校园内物品丢失后的信息不对称问题。

传统的失物招领方式存在以下痛点：
- 信息分散，丢失者和捡到者难以有效对接
- 联系方式暴露导致隐私泄露风险
- 时间地点描述不准确导致物品难以找回
- 缺乏有效的匹配机制

本项目通过数字化平台化方式，提供以下核心价值：
- 统一的失物招领信息发布与检索平台
- AI图像识别自动生成物品标签，提高匹配精度
- 关键词订阅与邮件通知，实现智能匹配推送
- 隐私保护机制隐藏真实联系方式

### 1.2 设计考虑

本设计说明书遵循以下原则：

1. **三层架构设计**：严格按UI层、业务逻辑层、数据层分离，便于维护和扩展
2. **RESTful API规范**：统一接口风格，便于前后端分离开发和接口测试
3. **安全性设计**：JWT Token认证、隐私保护、权限控制
4. **可扩展性设计**：模块化结构，支持后续功能迭代

### 1.3 读者对象

本文档主要面向以下读者：
- 项目开发团队成员（前端、后端）
- 项目评审专家
- 后续维护人员
- 软件工程实习指导教师

### 2.2 各层职责说明

#### 2.2.1 UI层（前端 Vue.js）

前端采用 Vue 3 + Vite 技术栈，主要职责包括：

- **视图展示**：渲染用户界面，展示帖子列表、聊天窗口等
- **用户交互**：处理用户输入、表单提交、按钮点击等交互事件
- **状态管理**：使用 Composition API 管理组件级状态
- **API调用**：通过 fetch 与后端 RESTful API 交互

主要视图组件：
- `HomeView`：首页大厅，包含帖子搜索、列表展示、发帖弹窗
- `SimpleHomeView`：简洁模式首页
- `ChatView`：即时通讯聊天界面
- `ProfileView`：个人中心，包含我的帖子、订阅管理、消息中心
- `AdminView`：管理员后台，包含仪表盘、帖子管理、日志审计

#### 2.2.2 业务逻辑层（后端 Spring Boot）

后端采用 Spring Boot 3.3.0 + MyBatis-Plus 技术栈，各层职责如下：

**Controller层（控制器）**：
- 接收和解析 HTTP 请求
- 参数校验和格式转换
- 调用 Service 层完成业务处理
- 返回统一格式的 JSON 响应

**Service层（服务）**：
- 核心业务逻辑实现
- 事务管理和数据一致性保证
- 复杂业务场景处理

**Mapper层（数据访问）**：
- 数据库 CRUD 操作
- MyBatis-Plus 提供的强大查询能力

#### 2.2.3 数据层（MySQL）

数据库采用 MySQL 8.0，共设计10张表：

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| app_user | 用户表 | id, username, password, email, nickname, avatar, role |
| item_post | 帖子表 | id, type, title, description, contact, imageUrl, buildingId |
| chat_session | 聊天会话表 | id, user1Id, user2Id, lastMsgId |
| chat_message | 聊天消息表 | id, sessionId, senderId, receiverId, content |
| app_comment | 评论表 | id, postId, publisherId, parentCommentId, rootCommentId |
| app_message | 站内消息表 | id, userId, type, title, content |
| post_subscription | 订阅规则表 | id, userId, keyword, buildingId |
| campus_building | 建筑字典表 | id, name, aliases, centerLat, centerLng |
| app_comment_like | 评论点赞表 | userId, commentId |
| admin_log | 管理员日志表 | id, adminId, actionType, targetId |

# 校园失物招领平台 数据库文档

## 概述

本项目为 **Campus Lost-Found** 校园失物招领平台，使用 Spring Boot + MyBatis-Plus + MySQL 开发。

---

## 1. 用户表 `app_user`

存储平台用户信息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | 账号（唯一） |
| password | VARCHAR(100) | NOT NULL | 加密密码 |
| email | VARCHAR(100) | UNIQUE | 邮箱（唯一） |
| nickname | VARCHAR(50) | | 昵称 |
| avatar | VARCHAR(255) | | 头像URL |
| role | VARCHAR(20) | DEFAULT 'USER' | 角色：USER / ADMIN |

---

## 2. 校园建筑字典表 `campus_building`

解决经纬度到建筑物的映射问题。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 建筑物ID |
| name | VARCHAR(100) | NOT NULL | 建筑物名称（如：西区第一食堂） |
| aliases | VARCHAR(255) | | 别名，逗号分隔（如：一食堂,西一） |
| center_lat | DOUBLE | | 中心纬度 |
| center_lng | DOUBLE | | 中心经度 |

**初始数据**：未来图书馆、聚英园、崇道楼、致知楼、崇智楼、弘雅堂、汇萃园

---

## 3. 失物招领帖子表 `item_post`

核心帖子表，支持区间时间查询和建筑物关联。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 帖子ID |
| type | VARCHAR(20) | NOT NULL | 类型：LOST(寻物) / FOUND(招领) |
| title | VARCHAR(255) | NOT NULL | 标题 |
| description | TEXT | | 详细描述（含AI标签） |
| contact | VARCHAR(100) | | 联系方式 |
| image_url | VARCHAR(255) | | 图片URL |
| item_status | VARCHAR(20) | DEFAULT 'PENDING' | 状态：PENDING / RESOLVED |
| publisher_id | VARCHAR(50) | NOT NULL | 发布者ID |
| create_time | DATETIME | NOT NULL | 系统发帖时间 |
| incident_start_date | DATE | | 可能发生的起始日期 |
| incident_end_date | DATE | | 可能发生的结束日期 |
| incident_time_desc | VARCHAR(100) | | 模糊时间描述（如：下午二节课后） |
| building_id | BIGINT | FK | 关联的建筑物ID（字典表FK） |
| location_desc | VARCHAR(255) | | 具体位置描述（如：二楼靠窗桌子） |
| latitude | DOUBLE | | 精确纬度（FOUND时可选） |
| longitude | DOUBLE | | 精确经度（FOUND时可选） |
| verify_question | VARCHAR(255) | | 验证问题（用于隐藏联系方式） |
| verify_answer | VARCHAR(255) | | 验证答案 |

---

## 4. 用户订阅规则表 `post_subscription`

用户订阅关键词和区域，用于智能匹配推送。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 订阅规则ID |
| user_id | BIGINT | NOT NULL | 订阅人ID |
| keyword | VARCHAR(100) | NOT NULL | 订阅关键词（如：AirPods, 黑色书包） |
| building_id | BIGINT | | 关注的区域（可选） |
| is_active | TINYINT(1) | DEFAULT 1 | 是否开启推送 |

---

## 5. 全局站内消息表 `app_message`

存储所有系统通知。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 消息ID |
| user_id | BIGINT | NOT NULL | 接收者ID |
| type | VARCHAR(20) | NOT NULL | 消息类型：SUBSCRIPTION / COMMENT / PRIVATE |
| title | VARCHAR(100) | NOT NULL | 消息标题 |
| content | TEXT | NOT NULL | 消息正文 |
| related_id | BIGINT | | 关联ID（帖子ID或私信ID，用于点击跳转） |
| is_read | TINYINT(1) | DEFAULT 0 | 是否已读（0-未读，1-已读） |
| create_time | DATETIME | NOT NULL | 发送时间 |

**索引**：`idx_user_read` (user_id, is_read)

---

## 6. 聊天会话表 `chat_session`

存储用户间的聊天会话。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 会话ID |
| user1_id | BIGINT | NOT NULL | 用户1ID |
| user2_id | BIGINT | NOT NULL | 用户2ID |
| last_msg_id | BIGINT | | 最后一条消息ID |
| update_time | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**唯一索引**：`uk_user_pair` (user1_id, user2_id)

---

## 7. 聊天消息表 `chat_message`

存储用户间的聊天消息。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 消息ID |
| session_id | BIGINT | NOT NULL | 会话ID |
| sender_id | BIGINT | NOT NULL | 发送者ID |
| receiver_id | BIGINT | NOT NULL | 接收者ID |
| content | VARCHAR(2000) | NOT NULL | 消息内容 |
| send_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 发送时间 |
| is_read | TINYINT | DEFAULT 0 | 是否已读（0-未读，1-已读） |

**索引**：`idx_session_id` (session_id), `idx_sender_receiver` (sender_id, receiver_id)

---

## 8. 帖子评论表 `app_comment`

评论与回复统一存储在同一表中。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 评论ID |
| publisher_id | BIGINT | NOT NULL | 评论发布者用户ID |
| post_id | BIGINT | NOT NULL | 所依附的帖子ID |
| is_reply | TINYINT(1) | DEFAULT 0 | 是否为回复：0-顶级评论，1-回复 |
| parent_comment_id | BIGINT | | 直接回复的那条评论ID |
| root_comment_id | BIGINT | | 所属顶级评论ID（便于归拢同一评论串） |
| content | TEXT | | 评论文字内容 |
| image_url | VARCHAR(255) | | 评论图片URL（最多一张） |
| create_time | DATETIME | NOT NULL | 发布时间 |
| like_count | INT | DEFAULT 0 | 点赞数 |

**索引**：
- `idx_comment_post` (post_id, create_time)
- `idx_comment_root` (root_comment_id, create_time)
- `idx_comment_parent` (parent_comment_id)

**设计说明**：
- 顶级评论：`is_reply=0`, `parent_comment_id=null`, `root_comment_id=自己ID`
- 回复评论：`is_reply=1`, `parent_comment_id=直接回复的评论ID`, `root_comment_id=顶级评论ID`

---

## 9. 评论点赞关系表 `app_comment_like`

多对多关系表，防止用户重复点赞。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| user_id | BIGINT | PK | 点赞用户ID |
| comment_id | BIGINT | PK | 被点赞评论ID |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 点赞时间 |

**索引**：`idx_comment_like_comment` (comment_id)

---

## 10. 管理员操作审计日志表 `admin_log`

记录管理员的关键操作。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 日志ID |
| admin_id | BIGINT | NOT NULL | 执行操作的管理员ID |
| action_type | VARCHAR(50) | NOT NULL | 操作类型（如：DELETE_POST） |
| target_id | VARCHAR(255) | | 被操作的对象ID（如帖子ID） |
| detail | TEXT | | 操作详情描述 |
| create_time | DATETIME | NOT NULL | 操作时间 |

---

## ER 关系简图

```
app_user (1) ──────< (N) item_post        # 用户发布帖子
app_user (1) ──────< (N) app_comment     # 用户评论
app_user (1) ──────< (N) app_message     # 用户接收消息
app_user (1) ──────< (N) post_subscription # 用户订阅
app_user (1) ──────< (N) chat_session    # 用户参与会话
app_user (1) ──────< (N) admin_log       # 管理员日志

campus_building (1) ──────< (N) item_post # 建筑物关联帖子
campus_building (1) ──────< (N) post_subscription # 建筑物订阅

item_post (1) ──────< (N) app_comment     # 帖子包含评论
app_comment (1) ──────< (N) app_comment_like # 评论被点赞
app_comment (N) ──────> (1) app_comment   # 回复关系（自关联）

chat_session (1) ──────< (N) chat_message # 会话包含消息
```
